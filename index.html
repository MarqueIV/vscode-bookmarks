Toggle & Label

Place a mark or name it—in one step.

Use Bookmarks: Toggle to mark or unmark any line instantly. Use Bookmarks: Toggle Labeled to attach a descriptive name that stays visible inline beside the code, so notes and TODOs live with the work without polluting source control.

Inline label visibility

Enable bookmarks.label.inline.enabled to show label text directly on the bookmarked line. Fine-tune font style, weight, margin, and color to fit your theme.

Smart label suggestions

Configure bookmarks.label.suggestion to pre-fill the label input from selected text or the entire line, reducing typing when you already have the right words highlighted.

Navigation

Move between what matters without searching.

Jump to the next or previous bookmark with Bookmarks: Jump to Next and Bookmarks: Jump to Previous. Navigation wraps around and crosses file boundaries so every marked position in the project is reachable in sequence.

Cross-file navigation

  • Navigate through all files that contain bookmarks
  • Wrap around at the first and last bookmark in scope
  • Use List from All Files for a searchable overview with line previews
  • Temporarily scrolls to a bookmark when selected in the list

Navigation controls

Limit scope to the current file with bookmarks.navigateThroughAllFiles, disable wrap-around with bookmarks.wrapNavigation, and choose whether the revealed line appears at the top or center of the editor.

Side Bar

A dedicated panel for every bookmark in the project.

The Bookmarks Side Bar organizes all marks by file, supports single-folder and multi-root workspaces, and gives you a persistent place to browse, jump, and manage bookmarks without opening the command palette.

Selection & Export

Turn bookmarks into selections or shareable documents.

Select Lines

  • Select all bookmarked lines in the current file at once
  • Particularly useful for working with log files
  • Combine with editor multi-cursor actions for bulk edits
  • Works with any file type

Expand & Shrink Selection

  • Expand selection to the next or previous bookmark
  • Shrink selection back toward the nearest mark
  • Compose region selections from multiple bookmarks
  • Ideal for analyzing structured blocks in log files or code

Export to Markdown

  • Export all bookmarks to a new Markdown document
  • Default format: a table grouped by file, sorted by line
  • Customize with bookmarks.export.pattern using $file, $line, $column, $label, and $content
  • Supports CSV, simple list, or any format you define

Persistence & Remote Development

  • Bookmarks are saved per session and restored on reopen
  • Save in project with bookmarks.saveBookmarksInProject to share via version control
  • Full Remote Development support: Docker, SSH, WSL, and Codespaces
  • Local bookmarks saved in project are available when connected remotely

FAQ

Quick answers for common questions.

How do I toggle a bookmark?

Place your cursor on any line and run Bookmarks: Toggle from the Command Palette or use the default keybinding. Running the command again on the same line removes the bookmark. You can also use Bookmarks: Toggle Labeled to attach a descriptive name at the same time.

Can I add labels to my bookmarks?

Yes. Use Bookmarks: Toggle Labeled to name a bookmark when you create it. You can configure bookmarks.label.inline.enabled to display label text directly on the bookmarked line in the editor, and configure bookmarks.label.suggestion to pre-fill the input from selected text or the current line.

How do bookmarks persist between sessions?

Bookmarks are automatically saved and restored each time you open the project. By default they are stored in VS Code's workspace state. If you set bookmarks.saveBookmarksInProject to true, bookmarks are saved in a .vscode/bookmarks.json file instead, making them portable across machines and shareable through version control.

Does Bookmarks work in remote development environments?

Yes. Bookmarks has full support for Remote Development scenarios including Docker containers, SSH targets, WSL, and GitHub Codespaces. When bookmarks.saveBookmarksInProject is enabled, bookmarks saved locally are also available when the same folder is opened remotely.

Can I export my bookmarks?

Yes. Use Bookmarks: Export to generate a new Markdown document with all bookmarks. The default format is a table grouped by file and sorted by line number. You can customize the output with bookmarks.export.pattern, using variables like $file, $line, $label, and $content to produce CSV, plain lists, or any format you need.

Does it work on VS Code forks like Cursor, Windsurf, and Antigravity?

Yes! It works on all VS Code forks, ensuring compatibility and seamless integration with your favorite tools.
